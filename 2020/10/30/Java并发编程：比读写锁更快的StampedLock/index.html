Why StampedLock?

读写锁已经在一定程度上对于读多写少的场景有了一定的并发优化，但是写锁的占用对于有读锁的存在的时候是被阻塞的，因此StampedLock提供了乐观读（无锁读）的方法，允许多个线程读的时候可以允许一个写线程占用写锁。

当StampedLock退化到这一步，其实就和原始的读写锁一样了

  • class StampedLockTest{
    final static Logger log = Logger.getLogger(StampedLock.class.getName());
    final static StampedLock lock = new StampedLock();
    final static Map<Integer,String> hashMap = new HashMap<Integer, String>();
    static {
        hashMap.put(1,"Mason");
        hashMap.put(2,"Jam");
    }
    public static String getInfo(int id) throws InterruptedException{
        /*
        加读锁后sleep两秒
        */
        long stamp = lock.readLock();
        log.info(Thread.currentThread().getName()+" get ReadLock");
        try{
            String name = hashMap.get(id);
            log.info(Thread.currentThread().getName()+" is blocking");
            Thread.sleep(2000);
            log.info(Thread.currentThread().getName()+" awake");
            return name;
        }finally {
            lock.unlockRead(stamp);
        }
    }
    public static void putInfo(int id,String info){
        long stamp = lock.writeLock();
        log.info(Thread.currentThread().getName()+" get WriteLock");
        try{
            hashMap.put(id,info);
        }finally {
            lock.unlockWrite(stamp);
        }
    }
}
public class Main {

    public static void main(String[] args) {
    /*
    一个线程读一个线程写
    */
        new Thread(()->{
            for(int i=0 ;i<10;i++){
                try{
                    StampedLockTest.getInfo(1);
                }catch (InterruptedException e){
                    e.printStackTrace();
                }
            }
        }).start();
        new Thread(()->{
            for(int i=0;i<10;i++){
                StampedLockTest.putInfo(1,"Test");
            }
        }).start();
    }
}

发现读线程blocking期间写线程依旧能进行。

More About “tryOptimisticRead”

极客专栏老师也有写到乐观锁在数据库中也有涉及，我也是有接触到过数据库中乐观读，主要是为表提供一个version列，每次读取数据的时候查也取出该行数据的version字段值，后面如果有依赖这行数据的其他数据需要写入的时候再次查看一下该行数据的version是否被其他线程给修改了，再进行数据的更新。

其实这边的version可以理解为上面代码中的stamp，都是用来表示数据是否被修改。
