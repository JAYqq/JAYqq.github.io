Java并发编程—同步工具类：CountDownLatch 和 CyclicBarrier

Posted on 2020-10-31 Edited on 2020-11-03 In Java Views: Valine:
《Java并发编程艺术mini》——并发编程基础、极客《Java并发编程实战》记录

What is Synchronize?

线程的同步是当不同线程之间存在协作关系，需要顺序执行时存在的，一般我们可以使用join来等待线程执行完后再执行，但是如果有很多线程，每个线程都进行join，这时候可以使用CountDownLatch

Instance of CountDownLatch

countDownLatch相当于设置了一个count为3的计数器，一个线程执行完后 countDown，主线程调用 await等待结束。

Application of CountDownLatch

  • 一个线程需要等待多个线程

CyclicBarrier

现在有个需求，就是有两个队列：生产队列和消费队列，两个线程不断生产数据到两个队列，当每分别生产一个后就需要取出队首比较一下大小。

其实使用计数器同样可以实现，计数器每减到0后就去对比，然后重新将计数器设置为2

CyclicBarrier帮我们做好了这种功能

Main Function

  • new CyclicBarrier 设置计数器大小和归零后执行的函数
  • await() 计数器减一

Application of CyclicBarrier

  • 适用于当前线程等待前面线程执行完后的计算结果，并需要进行整合
